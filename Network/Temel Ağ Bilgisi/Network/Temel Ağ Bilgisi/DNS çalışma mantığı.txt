🌐 DNS (Domain Name System) Çalışma Mantığı
Tanım:
DNS, internetin “telefon rehberi” gibidir.
İnsanların kolay hatırladığı alan adlarını (ör. google.com) bilgisayarların anlayacağı IP adreslerine çevirir.


📌 Çalışma Adımları
Kullanıcı bir alan adı yazar

Örn: Tarayıcıya www.example.com yazdın.

İlk sorgu yerel önbelleğe gider

Bilgisayar, tarayıcı veya işletim sisteminin DNS önbelleğinde bu adresin IP’si kayıtlıysa, direkt oradan alır.

Yerel DNS sunucusu (Resolver) devreye girer

Genellikle ISS (İnternet Servis Sağlayıcısı) tarafından sağlanır.

Yerel DNS sunucusunda da yoksa sorguyu internete yönlendirir.

Kök DNS sunucusu sorgulanır

Dünyada 13 ana kök sunucu vardır.

“Bu alan adı hangi uzantıya (.com, .org) ait?” diye sorulur.

TLD (Top Level Domain) sunucusu sorgulanır

Örn: .com uzantısının sorumlusu TLD sunuculara gidilir.

“example.com için yetkili DNS sunucusu nerede?” bilgisi alınır.

Yetkili (Authoritative) DNS sunucusu yanıt verir

İlgili alan adının IP adresi burada kayıtlıdır.

IP adresi kullanıcıya iletilir.

IP adresi önbelleğe kaydedilir

Sonraki sorgularda daha hızlı erişim sağlanır.



📌 Protokoller
DNS genellikle UDP 53 portunu kullanır (hızlı olduğu için).

Çok büyük sorgularda TCP 53 kullanılır.



📌 Siber Güvenlikte Önemi
DNS Spoofing / Cache Poisoning → Sahte DNS kayıtları ile kullanıcıyı yanlış siteye yönlendirme.

DNS Tunneling → DNS sorguları kullanarak veri sızdırma.

DNSSEC → DNS sorgularının dijital imzalarla doğrulanması, sahte kayıt riskini azaltır.


💡 Benzetme:

Alan adı (Domain) → Kişinin adı

IP adresi → O kişinin telefon numarası

DNS → Telefon rehberi
